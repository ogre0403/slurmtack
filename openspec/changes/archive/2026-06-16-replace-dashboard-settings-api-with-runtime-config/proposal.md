## Why

The dashboard is a browser application served as static HTML and JavaScript from nginx, so it cannot directly read the daemon container's `.env`. The current `GET /v1/dashboard/settings` route exists only to surface one safe value for a UI hint, which adds an unnecessary authenticated API contract for configuration that can be injected at page load.

## What Changes

- Add a dashboard runtime configuration file generated by the nginx container at startup and expose only a safe whitelist of UI settings through `window.SLURMTACK_CONFIG`.
- Move the dashboard's `SLURM_SIF_PATH` hint input from the authenticated `/v1/dashboard/settings` API to the runtime configuration file loaded before `dashboard.js`.
- Remove the dedicated dashboard settings API handler, response DTO, route registration, and the related Swagger/docs/test coverage.
- Update deployment wiring so the nginx container receives `SLURM_SIF_PATH` from `.env` and emits a non-cached runtime config asset without exposing secrets.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `node-switch-dashboard`: change the dashboard SIF hint contract to read safe runtime configuration at page load instead of fetching `/v1/dashboard/settings`.
- `swagger-generation`: remove the deleted `/v1/dashboard/settings` route from the documented public API surface.

## Impact

- Affected code: `docker/nginx/html`, nginx container startup/config wiring, `docker/docker-compose.yaml`, and `internal/api` server/handler/DTO/tests.
- Removed API: `GET /v1/dashboard/settings`.
- Updated documentation: dashboard behavior and generated Swagger artifacts.
